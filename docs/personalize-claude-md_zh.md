# 配置个人 CLAUDE.md

> [English](personalize-claude-md.md)

Muse 是一个整体助理——同时理解用户的身体状况、日常事务、财务状况、关心的人与生活全貌。它不会按话题切换人格，而是始终基于完整背景信息作出回应。

为实现这一点，Muse 启动时会读取归档根目录下的 `CLAUDE.md`。该文件是用户向 Muse 提供的**自传式简介**。

**重要说明：** 模板对所有人生阶段保持中性——学生、在职人员、自由职业者、全职父母、退休人士、创业者均适用。不适用的章节直接删除，无需强行填写。

---

## 如何生成此文件

### 对话引导（推荐）

点击顶部导航栏的「**整理档案**」按钮（curator 模式）。Muse 会开启一个会话，先扫一遍 archive 现状，再带你过 `CLAUDE.md` 里空着的章节——逐题提问、通过 `Edit` 将答案写回文件。任意章节可回复「跳过」或「先不填」。涉及隐私的章节（财务 / 健康）优先询问量级或定性描述，精确数字由用户自愿补充。

如果 `CLAUDE.md` 还不存在，顶 bar 会出现「未配档案」橙色提示，点击同样会跳到上述 workflow。

### 使用安装脚本（备选，交互性较弱）

`scripts/install-{linux,macos}` 首次运行时将询问是否：

1. 在 archive 下建好 6 个子目录（`health/` `work/` `money/` `people/`
   `notes/` `archives/`），每个带 README 说明
2. 做 7 个开放式 intake 问题：
   - 怎么称呼你
   - 出生年份（或年龄段）
   - 现住地
   - 一周大部分时间在做什么（学习 / 工作 / 自由职业 / 照护 / 退休 ……）
   - 一句话当前人生阶段
   - 今年主目标
   - 当前最关心的健康问题
3. 把答案直接 patch 进 CLAUDE.md 对应字段
4. 提示你接下来该把哪些原件丢进哪个目录

跳过初始问卷也可以，所有问题按 Enter 跳过即可。后续可随时通过顶 bar 的「整理档案」按钮回来继续填写。

### 手动建

```bash
cp scripts/templates/default-CLAUDE.md ~/muselab-archive/CLAUDE.md
sed -i "s/%DATE%/$(date +%Y-%m-%d)/" ~/muselab-archive/CLAUDE.md
# 子目录骨架
cp -r scripts/templates/archive-skeleton/* ~/muselab-archive/
```

## 模板的 9 个章节

| 章节 | 填写内容 |
|---------|--------|
| **1. 我是谁** | 称呼 / 年龄 / 城市 / 语言 / 居住情况 |
| **2. 我现在主要在做什么** | 一句话描述当前人生阶段 + 按角色选填的状态（学生 / 在职 / 自雇 / 照护 / 退休……） |
| **3. 财务** | 收入来源 / 量级 / 主要关注点（不限人生阶段） |
| **4. 身体** | 基本情况 / 体检 / 用药 / 关注点 |
| **5. 我关心的人** | 不只是家人——伴侣 / 父母 / 朋友 / 任何重要的人 |
| **6. 当前所思所想** | 近期烦恼 / 正在进行的事 / 想做但还未开始的事 |
| **7. 归档目录内容** | 6 个子目录索引 + 当前已有的关键材料 |
| **8. Muse 的协作方式** | 行为约定（中性，适用于所有人） |
| **9. 需要自行维护的内容** | 生活变化时应更新哪些章节 |

## 子目录结构（6 个，通用于所有人群）

| 目录 | 适合存放 | 学生 | 在职 | 自由职业 | 全职父母 | 退休 |
|------|--------|------|------|---------|---------|------|
| `health/` | 身体相关 | 入学体检 | 年度体检 | 同左 | 自己及子女 | 慢病管理 |
| `work/` | 当前从事的事务 | 论文/申请材料 | 简历/项目 | 作品集/客户 | 育儿记录 | 当前活动 |
| `money/` | 财务相关 | 月度预算 | 收支/储蓄 | 报税/应急资金 | 家庭预算 | 现金流 |
| `people/` | 关心的人 | 父母/朋友 | 伴侣/同事/父母 | 同左 | 配偶/子女/双方父母 | 伴侣/子女/老友 |
| `notes/` | 杂项笔记 | 通用 | 通用 | 通用 | 通用 | 通用 |
| `archives/` | 原件归档 | 通用 | 通用 | 通用 | 通用 | 通用 |

每个目录均附有各自的 `README.md`，包含针对不同人生阶段的具体建议。

## 关键设计原则

### Muse 是一个整体助理，而非多个角色

跨领域决策是 Muse 的核心价值所在。示例：**父母刚完成心脏支架手术 + 用户今年的现金流状况 + 未来几年是否计划换工作 → 是否应为父母升级香港医疗保险？** 多角色模型会将此拆分给三个独立专家，产生三套互不连贯的建议；统一助理才能给出真正契合用户实际情况的综合方案。

### 行为约定属于 Muse，而非用户的填写义务

§8 中"涉及健康 / 涉及财务 / 涉及学业或工作"的规则是 **Muse 的回应承诺**（例如：涉及健康必须引用指南、不做诊断），而非要求用户填写所有相关章节。

## 维护节奏

| 触发时机 | 更新位置 |
|------|--------|
| 体检后 | §4，PDF 放入 `health/` |
| 学业 / 工作 / 业务变化 | §2 |
| 重大财务变化 | §3，记录放入 `money/` |
| 关心的人发生重大变化 | §5 |
| 定期回顾 | 每半年回顾一次，删除不再适用的内容 |

## 隐私与安全

- 强烈建议为 muselab 归档目录启用文件系统加密（macOS FileVault / Linux LUKS）。
- 不要将归档目录同步至 OneDrive / Google Drive / Dropbox 等公共云存储。
- 涉及他人的信息可进行脱敏处理（以"父亲"/"M"代替真实姓名）。
- 密码、身份证号、银行账号请存入专用密码管理器，不要写入归档目录。
- 远程备份请使用 [restic](https://restic.net) 或 [borg](https://borgbackup.org)，并启用端到端加密。
