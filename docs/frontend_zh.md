# 前端内部机制

> [English](frontend.md)

本文介绍 muselab 浏览器端的结构、状态管理与性能策略。PWA 安装见
[手机端](mobile_zh.md)，真实终端见 [终端](terminal_zh.md)，服务端流式协议见
[模型路由与 chat 循环](routing_zh.md)。

## 1. 无构建前端

muselab 使用原生 HTML、CSS、JavaScript 和 Alpine.js，不依赖 Webpack、Vite
或 Rollup。开发时修改 `frontend/` 后刷新浏览器即可。

启动阶段只加载基础运行库。highlight.js 与 KaTeX 在浏览器空闲时预热，
CodeMirror 在编辑交互前预热，Mermaid 与 xterm.js 保持按需加载。所有第三方
资源都随项目提供，不依赖 CDN。

静态资源 URL 带版本标记；Service Worker 从 `/sw.js` 提供服务，使作用域覆盖
整个站点。服务器会缓存渲染后的首页，并在前端资源版本变化后使缓存失效。

## 2. 工作台与状态

桌面端采用三栏布局：

| 区域 | 主要能力 |
|---|---|
| 文件区 | 多工作区、文件树、最近打开、上传、搜索、回收站 |
| 预览区 | 文件标签、真实终端标签、预览、查找、编辑与 Markdown 分屏 |
| 会话区 | 多会话标签、消息流、队列、附件、模型与权限控制 |

移动端复用同一套状态，通过底部导航在文件、预览、会话三个页面间切换。会话区
顶栏的终端按钮会切换到预览页并打开完整终端管理器。

每个会话标签保存自己的消息、输入草稿、附件、滚动位置、流式状态、服务端队列
镜像和 Footer 状态。切换标签不会重置正在运行的任务；后台标签完成后会显示未读
状态。全局任务中心汇总不同工作区和会话中的运行、等待确认、失败及未读任务。

## 3. 文件预览与编辑

预览区支持 Markdown、文本、HTML、图片、PDF、XLSX 和 CSV。HTML 在沙盒
iframe 中运行，不能读取 muselab 页面、浏览器存储或鉴权信息。

Markdown 和文本文件可以进入编辑模式。Markdown 编辑器提供纯编辑、分屏和
预览三种视图，并显示保存状态。文件内查找支持匹配导航和上下文结果；全局搜索
用于跨文件、跨会话定位内容。

文件标签与终端标签共用预览区标签栏。终端进程不会因为切换到文件预览而停止。

## 4. 消息渲染

助手 Markdown 的主要流程为：

```text
流式文本 → Markdown 解析 → HTML 净化 → 最终数学公式与文件路径增强
```

流式阶段跳过昂贵的 KaTeX 和文件路径遍历，回合结束后再执行完整渲染。
AI 生成的 HTML 产物和文件 HTML 预览都在不含 `allow-same-origin` 的沙盒中
运行。

桌面端优先保证阅读体验：

- 小于 32 KiB 的当前文本片段约每 80 ms 更新；
- 32–128 KiB 约每 160 ms 更新；
- 更长片段约每 320 ms 更新；
- 保留 4 个消息面板，常规展示 60 个气泡，DOM 上限 300，内存上限 800。

移动端优先控制耗电、内存和 WebView 卡顿：

- 根据长度逐步使用 80、160、320、600、1,000、1,600 ms 的刷新间隔；
- 流式片段超过 32 KiB 后临时改用纯文本，回合结束时再生成完整富文本；
- 只保留当前消息面板，常规展示 15 个气泡，DOM 上限 60，内存上限 120；
- 不在后台预加载其他会话，较早记录按 80 条一批加载。

用户在流式消息中选择文本时，前端会暂停替换对应 HTML，避免选择范围被新 token
打断。

## 5. SSE 会话流

新回合分两步建立：

1. 浏览器通过 `POST /api/chat/stream/start` 发送 prompt、会话、模型、权限和附件，
   并使用 `X-Auth-Token` 鉴权。
2. 服务端返回短时一次性 ticket，浏览器仅以该 ticket 打开
   `/api/chat/stream` 的 `EventSource`。

因此 prompt 和长期 token 不会出现在 SSE URL、浏览器历史或常规访问日志中。
只有连接旧版后端、明确返回 404／405 时才使用旧协议。

客户端处理文本、思考、工具调用、后台任务、限速、提问、权限确认、完成、取消和
重同步等事件。服务端每 15 秒发送心跳；连接完全静默超过 40 秒时，客户端通过
活动探测和磁盘回放恢复，不会重新执行已完成的回合。

服务端消息队列独立于页面连接。用户可在 Muse 回复期间继续发送，队列会按顺序
执行；页面关闭后仍可继续。出错、停止、提问或等待授权时队列会暂停，避免无人
值守地继续扩大影响。

## 6. Footer 与 Context

会话 Footer 只放与当前回合直接相关的控制：附件、模型、权限、effort 和 context
占用。运行时长以服务端回合开始时间为基准，因此重连和切换标签不会归零。

Context 环可展开查看 SDK 返回的上下文组成，并提供原生 compact 能力。Context
接近上限时 UI 会提示风险，但最终模型限制以 Provider 和 SDK 返回为准。

Skills、MCP、终端、图像生成和全局任务中心属于会话或工作台级能力，放在顶栏而
不是 Footer，避免移动端输入区拥挤。

## 7. 终端前端

预览顶栏的终端按钮紧邻内容搜索入口，打开终端列表和 Profile 选择器。终端使用
xterm.js，按需加载；桌面端 scrollback 为 10,000 行，移动端为 3,000 行。

移动端通过主动触摸手势驱动 xterm scrollback，并提供复制、粘贴、反斜杠、
Ctrl+C、Esc、Tab 和方向键。完整生命周期、Profile 和安全说明见
[终端](terminal_zh.md)。

## 8. PWA 与 Service Worker

Service Worker 只负责 Web Push，不缓存页面或静态资源。收到推送时，如果已有
可见的 muselab 窗口，会抑制系统通知。

PWA 从后台恢复时会比较资源版本；发现版本变化且当前没有流式任务时自动重载，
避免旧 JavaScript 与新后端不兼容。移动安装、HTTPS 和通知限制见
[手机端](mobile_zh.md)。

## 9. 国际化与无障碍

界面内置简体中文和英文，默认根据浏览器语言选择，并可在设置中覆盖。翻译缺失时
依次回退到中文和键名。

交互控件保留键盘和触摸路径；输入框在移动端不低于 16 px，支持安全区、减少动画
偏好和软键盘视口变化。
