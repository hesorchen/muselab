# 手机端（PWA）

> [English](mobile.md)

muselab 自带 Web App Manifest 和主屏图标，可以作为 PWA 安装到 iOS 和
Android。它与桌面端使用同一套代码和后端，不需要 `.ipa`、`.apk` 或应用商店。

## 安装

PWA、Service Worker 和 Web Push 都要求 HTTPS。裸
`http://192.168.x.x:PORT` 在 iOS 上不属于安全上下文。

可使用以下任一方式：

1. 通过 Tailscale 的 `*.ts.net` HTTPS 地址访问；
2. 使用真实域名和证书，并运行 `scripts/setup-https.sh` 配置反向代理。

iPhone 必须用 Safari 打开页面，然后选择“分享 → 添加到主屏幕”。Android
Chrome 通常会直接提供“安装”入口。

## 移动布局

底部导航在文件、预览和会话之间切换。每个页面都有独立滚动容器，系统下拉刷新
被禁用，以免误触中断会话；文件树在顶部保留自己的下拉同步手势。

会话顶栏包含终端快捷按钮。点击后会切到预览页并打开终端管理器，二级窗口顶部
提供“新建终端”，下面可选择普通 Shell 或指定 Profile。

软键盘弹出时，输入框与当前会话保持可见。触屏输入使用至少 16 px 字号，避免
iOS 自动缩放；底部工具栏适配安全区。

## 移动端会话性能

移动端只挂载当前会话的消息面板，不在后台预加载其他会话。默认展示最近 15 个
气泡，DOM 上限 60，内存消息上限 120，较早历史每次加载 80 条。

流式 Markdown 会随文本增长逐步降低刷新频率。单个流式片段超过 32 KiB 后先以
纯文本显示，结束时再完成 Markdown、数学公式和路径增强，从而减少长回复导致的
发热、掉帧和软键盘卡顿。

会话队列、当前回合计时、滚动位置、输入草稿和 Footer 状态都按标签保存。切换
页面或恢复 PWA 不会重置正在运行的回合。

## 移动终端

终端在预览区中使用真实 PTY。上下拖动终端内容会滚动历史，而不会带动整个页面；
移动端最多保留 3,000 行 scrollback，以限制内存占用。

终端底部提供横向可滚动的扩展键：

- 复制、粘贴和反斜杠 `\`；
- Ctrl+C、Esc、Tab；
- 上、下、左、右方向键。

这解决了部分移动键盘难以输入反斜杠和控制键的问题。终端断线时会自动申请新的
一次性 ticket 并重连；切换页面不会停止终端进程。详见
[终端](terminal_zh.md)。

## Web Push

在“设置 → 通知”中启用。VAPID 密钥首次启动时由服务器生成并保存到
`<主工作区>/.muselab/vapid.json`；设备订阅保存在
`<主工作区>/.muselab/push_subs.json`，而不是 `.env` 或仅存在浏览器本地。

浏览器仍保存自己的 PushSubscription，并将其登记到服务器。任务完成、队列暂停
或其他后台事件可在页面关闭时通知设备；点击通知会打开对应工作区和会话。
如果 muselab 已有可见窗口，系统通知会被抑制。

### iOS 限制

- 必须先添加到主屏幕，再从主屏幕启动并开启通知；
- 普通 Safari 标签页不能为该站点启用 PWA Web Push；
- iOS Safari 忽略网页震动请求，但通知仍会出现。

不要随意删除 `vapid.json`。密钥变化会使所有现有订阅失效，需要每台设备重新
订阅。

## 版本恢复

PWA 从后台恢复时会检查前后端资源版本。版本变化且当前没有流式任务时会自动
重载，避免旧页面与新服务不兼容。如果界面长时间停留在旧版本，可完全关闭 PWA
后重新打开。
