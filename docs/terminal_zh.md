# 终端

> [English](terminal.md)

muselab 在预览区提供真实 Unix PTY。它不是命令模拟器：用户输入会以 muselab
服务账户的权限在当前工作区执行。

## 使用入口

桌面端在预览顶栏点击 `>_`，即可打开当前工作区的终端管理器。移动端也可以从
会话顶栏点击同一入口，界面会自动切到预览页。

终端管理器支持：

- 新建、打开、重命名和关闭终端；
- 查看运行或退出状态、Profile 和当前目录；
- 关闭当前工作区中的全部终端；
- 在预览标签栏中与文件标签切换。

每个终端绑定创建时的工作区，默认工作目录就是该工作区根目录。切换标签、页面或
会话不会终止进程；显式关闭终端会终止其中的进程。

## Profile

Profile 是“名称＋启动命令”的持久化模板。新建终端时可以选择：

- 普通 Shell：只启动交互式 Shell；
- 指定 Profile：Shell 启动后自动输入并执行保存的命令。

Profile 可以新建、编辑、删除，并可将一个 Profile 设为默认。界面会记住最近
选择；每次点击“新建终端”时以选择器当下的值为准。删除 Profile 不会影响已经
运行的终端。

Profile 存储在主工作区的
`<主工作区>/.muselab/terminal_profiles.json`，适合纳入备份。Profile 命令是
可执行配置，应按脚本或凭据同等谨慎地管理，不要写入密钥。

## 连接与恢复

终端连接分两步建立：

1. 浏览器使用 `X-Auth-Token` 请求短时一次性 ticket；
2. 浏览器把 ticket 作为 WebSocket 子协议连接终端。

长期 token 不进入 WebSocket URL。ticket 有效期为 30 秒且只能使用一次，服务端
还会校验浏览器 Origin。

终端在服务端保留有限输出缓冲。浏览器重连后会回放仍在缓冲中的输出；如果更早
内容已经被截断，终端会显示提示。页面断开不会立刻停止进程，超过 detached TTL
后才由服务端回收。回放期间，前端不会把历史设备查询重新产生的终端响应发送给
当前进程，以免出现 `0;276;0c` 一类控制序列残片。已退出终端也会在 exited TTL
后清理。

## 输入与 scrollback

xterm.js 只在首次打开终端时加载。

| 设备 | scrollback | 操作重点 |
|---|---:|---|
| 桌面端 | 10,000 行 | 键盘输入、鼠标滚轮、常用复制粘贴快捷键 |
| 移动端 | 3,000 行 | 单指上下拖动历史、扩展键、较低内存占用 |

桌面端支持平台复制粘贴快捷键，以及 Ctrl+Shift+C／Ctrl+Shift+V。移动端底部
提供复制、粘贴、反斜杠、Ctrl+C、Esc、Tab 和方向键。竖直触摸手势由终端主动
转换成逐行滚动，因此不会把整个预览页一起拖动。

## 配置

| 环境变量 | 默认值 | 含义 |
|---|---:|---|
| `MUSELAB_TERMINAL_ENABLED` | `1` | 是否启用终端 |
| `MUSELAB_TERMINAL_SHELL` | 自动检测 | Shell 可执行文件 |
| `MUSELAB_TERMINAL_MAX_SESSIONS` | `8` | 全服务同时运行的终端上限 |
| `MUSELAB_TERMINAL_BUFFER_BYTES` | `2097152` | 每个终端的服务端回放缓冲 |
| `MUSELAB_TERMINAL_DETACHED_TTL` | `1800` | 无客户端连接的运行终端保留秒数 |
| `MUSELAB_TERMINAL_EXITED_TTL` | `3600` | 已退出终端记录保留秒数 |

环境变量的边界和完整配置以 [配置参考](configuration_zh.md) 为准。

## 安全边界

终端拥有 muselab 服务账户能够访问的文件、网络和进程权限，通常不受工作区目录
边界限制。将 `MUSELAB_TOKEN` 交给他人，等价于让其通过浏览器使用该服务账户的
Shell。

终端子进程只继承必要的基础环境变量，不继承 `MUSELAB_TOKEN` 或 Provider API
密钥。不过，服务账户本身可读取的配置文件仍可能包含敏感信息，因此应：

- 只通过 HTTPS 或可信私网访问；
- 使用独立低权限账户运行 muselab；
- 不把服务暴露给不受信任用户；
- 不以 root 身份运行；
- 对 Profile 命令和备份文件采用与脚本相同的审查标准。

更完整的边界见 [安全模型](backend-security_zh.md)。

## 常见问题

- 找不到“新建终端”：先打开 `>_` 终端管理器；按钮在二级窗口顶部。
- 选了 Profile 却启动了其他命令：确认选择器已显示目标 Profile，再创建；如仍
  异常，刷新页面后重试。
- 移动端无法滚动：在终端正文内单指竖直拖动，不要从底部扩展键条开始。
- 无法输入反斜杠：使用扩展键条中的 `\`。
- 点击或滑动后出现 `2;276;` 一类字符：刷新旧页面；这是残留的鼠标坐标上报，
  新版前端会在普通 Shell 缓冲区自动拦截并复位。
- 达到数量上限：关闭不再使用的运行终端，或调整
  `MUSELAB_TERMINAL_MAX_SESSIONS` 后重启。
