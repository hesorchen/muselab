CORE IDEA

不是 RAG，不是云端拷贝，
就是磁盘上的原文件。

muselab 不预先向量化你的档案。Muse 通过 Read / Grep / Edit 工具按需访问——和你自己在 IDE 里翻文件一样直接。文件改了立即生效，没有索引同步，没有 embedding 漂移。你的档案始终是你能直接打开、修改、删除的原文件，不是某个服务里的黑盒索引。

PRIVACY · 隐私与数据主权

私密性不是开关，
是架构默认。

muselab 是单用户、自托管的。没有遥测、没有云端副本、没有第三方账号——文件在你的磁盘上，密钥在你的 .env 里，连 Muse 自己都被禁止读取凭证文件。

零遥测
不接任何 analytics，不回传使用数据。除了你主动发起的模型 API 调用，muselab 不与任何第三方服务器通信。
连密钥都对 Muse 不可见
.env · SSH 私钥 · *.pem · credentials.json——这类凭证文件即便持有有效 token 也被拒绝读写。
文件就在你的磁盘上
archive 是你机器上一个普通目录。没有向量库、没有云端镜像、没有数据搬运——随时能用文件管理器或终端直接打开。
越权即拒
所有读写被锁在 archive 根目录内；指向 /、/etc 等系统路径会在启动时直接拒绝。
不被搜索引擎收录
默认 noindex + robots.txt，万一误暴露也不会被爬；token 用 constant-time 比较，过短拒绝启动。
MIT 开源，可审计
全部代码公开。每一条数据流向哪里，你都能自己读源码确认，而不是信一句"我们很安全"。

AGENT SDK · 完整能力面

不只是聊天框，
是完整的 agent。

muselab 由 Claude Agent SDK 直接驱动——Skills、MCP、Subagent、plan 模式都是原生能力，且在八家模型上行为一致。不是套壳聊天，是能调工具、能拆任务、能扩展的工作 agent。

Skills · 11 个预置技能包
模型按任务自动选用——web-search、code-reviewer、mermaid-helper、csv-analyzer、pptx、translate、meeting-notes…… 放一个 SKILL.md 进目录，就能加你自己的。
MCP · 接外部工具与数据源
通过 Model Context Protocol 接入 filesystem、git、fetch、memory、time、sequential-thinking 等 server，能力随开放生态持续扩展。
Subagent · 拆分子任务并行
复杂任务派生出独立上下文的子 agent 并行处理，各自跑工具链再汇总——主线程不被一次性塞满。
skills="all" 启动时自动发现 · MCP server 配置见 mcp.json · plan 模式见下一节

FAQ · 常见问题

开工前，
你可能想问的几件事。

需要 API key 吗？
Claude 走一次 claude login 的 OAuth，复用你的 Pro / Max 订阅——不需要 API key，也不按 token 另计费。其他七家各填自己的 API key，在设置里粘贴即可。
muselab 自己收费吗？
不收。MIT 开源，完全免费。模型调用直连各厂商、走你自己的订阅或 key——muselab 不中转、不加价（后端无代理层）。
我的数据存在哪？
全部在你的机器上：archive 是一个本地目录，对话记录是本地 JSONL 文件。没有云端副本，没有第三方账号，没有数据搬运。
能接本地模型 / 完全离线吗？
后端是 Claude Agent SDK，需要 Anthropic 兼容端点。目前不内置 Ollama 等纯本地推理；但模型 base_url 可在会话级覆盖，能指向任何兼容 Anthropic 协议的端点，包括你自建的代理。
怎么升级和卸载？
升级跑 upgrade.sh，卸载跑 uninstall-linux.sh / uninstall-macos.sh。卸载只移除系统服务——你的 .env、对话记录和 archive 文件原样保留。
手机上怎么用？
和桌面访问同一台服务器、同一个会话。窄屏自动折叠成「文件 / 预览 / Muse」三 tab 单栏，还能作为 PWA 添加到主屏。
